# Telegram Транскрибатор Бот

Полнофункциональный Telegram-бот для транскрибации аудиофайлов и голосовых сообщений в текст с использованием мощной нейросети AssemblyAI.
Написан на асинхронном фреймворке `aiogram v3`.

## Возможности
- 🎧 Принимает любые голосовые сообщения и популярные форматы аудиофайлов (.mp3, .ogg, .wav и т.д.)
- 🌐 Выбор языка транскрибации или автоопределение.
- 🗣 Опциональное распознавание и разделение разных спикеров (говорящих) в аудио.
- 📄 Отправка результата прямо в чат (с разметкой) или `.txt` файлом, если текст длинный.
- 🔄 **Бот работает в бесплатном режиме** — без ограничений по балансу.

## Локальный запуск

1. Клонируйте репозиторий:
   ```bash
   git clone <URL_РЕПОЗИТОРИЯ>
   cd "Telegram Транскрибатор"
   ```

2. Создайте виртуальное окружение и установите зависимости:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```

3. Настройте конфигурацию:
   Скопируйте пример файла конфигурации:
   ```bash
   cp .env.example .env
   ```
   Отредактируйте `.env` и вставьте ваши ключи:
   - `BOT_TOKEN` — получаем у [@BotFather](https://t.me/BotFather)
   - `ASSEMBLYAI_API_KEY` — ключ от [AssemblyAI](https://www.assemblyai.com/)

4. Запустите бота:
   ```bash
   python3 main.py
   ```

## Деплой на сервер с помощью Docker

Проект содержит `Dockerfile` и `docker-compose.yml` для быстрой упаковки и деплоя на любом VDS/VPS сервере.

1. Установите Docker и Docker Compose на вашем сервере.
2. Скопируйте проект на сервер.
3. Отредактируйте `.env` файл, вписав реальные ключи.
4. Запустите контейнеры в фоновом режиме:
   ```bash
   docker-compose up -d --build
   ```

Бот запустится автоматически и будет работать в фоновом режиме. Для просмотра логов:
```bash
docker-compose logs -f
```

## Как устроен проект

- `main.py` — Точка входа, запуск бота, настройка меню и описания.
- `config.py` — Загрузка ключей окружения.
- `debug.py` — Утилита для отладки и диагностики бота.
- `handlers/user_handlers.py` — Логика общения бота с пользователем, прием аудио и выдача текста.
- `handlers/states.py` — FSM состояния (запоминание аудиофайла до выбора языка).
- `keyboards/inline.py` — Создание инлайн клавиатур.
- `services/transcriber.py` — Интеграция с сервисом транскрибации AssemblyAI.
- `database.py` — SQLite база данных для хранения информации о пользователях.
